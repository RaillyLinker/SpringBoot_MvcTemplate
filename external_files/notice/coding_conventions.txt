[Coding Conventions]
- external_files/notice 디렉토리는, 개발자들 사이에 공유해야할 사항을 적는 텍스트 파일을 모아두는 디렉토리이며,
    coding_conventions.txt 파일은 코딩 규칙을,
    TO_DO.txt 파일은 프로젝트 단위로 처리해야 하는 작업을 공지하고 확인하면 됩니다.

- src 와 동일한 위치의 외곽 디렉토리 중,
    by_product_files 라는 디렉토리는 프로젝트 런타임에 생성되는 파일을 저장하는 디렉토리입니다.
    동일 위치의 external_files 라는 디렉토리는 개발자가 임의로 넣어주는 파일들 저장하는 디렉토리입니다.
    이와 같은 구분을 통하여 프로젝트 기본 생성 디렉토리와 개발자의 의도로 만든 디렉토리를 구분하였습니다.

- 날짜 데이터 관련하여 다루는 기본 형태는 yyyy_MM_dd_'T'_HH_mm_ss_SSS_z 의 형태로,
    위와 같은 형태를 사용하면 밀리초 3자리 까지의 정밀도와 시차 관련 안정성을 확보할 수 있습니다.

- 본 프로젝트에서 자동 로깅 필터는 multipart/form-data 형식의 Request Body 는 로깅하지 않습니다.
    이 경우에는 수동으로 로깅 처리를 해주세요.
